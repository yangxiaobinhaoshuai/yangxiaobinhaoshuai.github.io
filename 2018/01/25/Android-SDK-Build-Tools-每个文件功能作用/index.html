Android SDK Build Tools 每个文件功能作用

发表于 2018-01-25

Android SDK Build Tools

google 官网的release note 最新version 为27.03 2018.1.25

Introduction

官网简介为：Anbroid sdk build tools 是一个构建Android app 时候的必要的组件。 其实也就是Android app 由我们ide中的源码编译成apk 这一系列过程中所需要的构建工具。

你应该使你的sdk build tools 一直保持和 Android sdk manager 同样是最新版。如果你使用了gradle plugin 3.0或更高的版本， 你的项目会默认使用一个特定版本的sdk build tools。 如果你想使用其他版本的sdk build tools，用project 的build.gradle 中的buildToolsVersion 字段来指定其他版本。

Detail

  1. aapt
    1. 编译source 资源为二进制
  2. aapt2
    1. aapt2 编译资源 增量编译
    2. 不适合使用aapt2的场景：
      1. 插件化，和热修复中，需要使用public.xml 的场景
      2. 构建过程，需要动态增删资源的场景，如删除一部分线上不应该出现的资源
  3. aarch64-linux-android-ld
  4. aidl
  5. apksinger
  6. arm-linux-androideabi-ld
  7. bcc_compat
  8. dexdump
    1. dexdump 是android 提供的一个dex文件查看工具，dexdump 会按照类的定义，清晰的列出类的类结构和内容
  9. dx
  10. I686-linux-android-ld
  11. Jack and jill 系列
  12. lib
    1. apksigner
    2. dx.jar
    3. shrinkedAndroid.jar
  13. lib64
    1. libaapt_jni.dylib
    2. libbcc.dylib
    3. libbcinfo.dylib
    4. libc++.dylib
    5. libclang.dylib
    6. libLLVM.dylib
  14. llvm-rs-cc
  15. mainDexClasses
  16. mainDexClasses.rules
  17. mainDexClassesoApt.rules
  18. mipsel-linux-android-ld
  19. NOTICE.txt
  20. package.xml
  21. randerscript
    1. 略
  22. runtime.properties
  23. source.properties
  24. split-select
  25. X86_64-linux-android-ld
  26. zipalign
