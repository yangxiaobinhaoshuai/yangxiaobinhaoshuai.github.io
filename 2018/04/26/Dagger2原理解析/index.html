Dagger2简介

Dagger2 是一个全静态、编译时 的依赖注入 framework， 可用于 Java 和 Android。是 Google fork 自 Square 的 Dagger 项目。

Github 首页

官方文档

本文旨在介绍 Dagger2 原理以及 Dagger2 使用时候的权衡。

Why Dagger2？

老实说，Dagger2 学习曲线稍显陡峭。而且以注解的方式和对象实例化打交道，离开了文档说明，更显的拒人于千里之外。但是每个 App 甚至每个程序都需要去实例化各式各样的对象来完成功能，无论是面向对象的思想还是高内聚低耦合的类设计准则，都离不开对象。

在使用对象过程中有几点注意项：

  1. 对象的生命周期尤为重要，有时候我们需要手动用工厂或者单例模式来保证对象实例正常工作和节省内存资源。
  2. 当想对象中传递依赖时候，通常我们会构造传参，当需要新的依赖或者删除某些依赖的时候，这个对象的构造器会变化，那么实例化这么对象的地方都需要对应修改。

通常，在使用过程中，时常会令我困惑：

  1. 当对象依赖关系复杂的时候，实例化我想需要的对象，总会莫名其妙的实例化一堆我不需要的对象。
  2. 在不同的功能模块中，需要同一对象不同实例的时候，因为命名相似总会搞混而花费时间 debug 排查。
  3. 在 Android 中，内存很紧俏，单例和重用对象显得很重要，但是当程序设计或者逻辑不是很明朗的时候，我会 new 很多不必要的对象，或者滥用单例造成内存泄漏。

所以，我希望对象的实例化更可控，更有条理，在程序运行中对象的实例像数据库的表一样，每个对象有它的作用域和生命周期。当我实例化某个对象的时候，不需要考虑它内部的依赖传递关系。在既保证逻辑正常执行的同时，又不过分浪费内存。这样更能清晰描述对象和功能模块之间的联系，耦合性更低，可读性更强，更容易理解业务逻辑。所以当我听说了 Dependency Injection 的概念非常感兴趣。
