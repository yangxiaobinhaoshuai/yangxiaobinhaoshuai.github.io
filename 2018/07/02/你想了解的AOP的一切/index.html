关于 AOP 的一切

编程思想概念

  • AOP ：Aspect-Oriented Programming 面向切面编程 （ APT、ASM、Aspectj ）
  • OOP：Object-Oriented Programming 面向对象编程 （ Java、C++、Objective-C）
  • POP : Procedure-Oriented Programming 面向过程编程 （C）

####编程思想异同

  • 面向过程 以事件为中心，顺序调用，更注重每个环节的功能。
  • 面向对象 更注重封装和把复杂的问题用现实实际的思维简化。
  • 面向切面 感觉更注重解耦，在不修改源码前提下，动态为多个业务模块拓展新功能。

  无论面向什么，只是我们用程序去解决实际问题时候的不同角度而已。面向对象将程序划分角色，每个角色负责一项功能，像我们人类生活一样，抽象复杂问题模型的时候，运用这种思想更友好些；而面向切面，个人理解就是看待的对象不再是实际问题，而是现有的程序，将一个整体的程序按某种逻辑划分几个模块，然后再通过编写AOP程序去为划分的模块做拓展。

AOP切入时机

总之，AOP 是以通过编译期或者运行期动态代理，在不修改源码情况下，实现统一拓展与维护为目的。那么，从程序源码编译，到映射到内存形成对象开始工作之前的任意环节进行切入处理都可以～ ，做了张图
