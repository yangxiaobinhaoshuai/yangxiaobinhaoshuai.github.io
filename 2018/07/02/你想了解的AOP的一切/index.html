关于 AOP

编程思想概念

  • AOP ：Aspect-Oriented Programming 面向切面编程 （ APT、ASM、Aspectj ）
  • OOP：Object-Oriented Programming 面向对象编程 （ Java、C++、Objective-C）
  • POP : Procedure-Oriented Programming 面向过程编程 （C）

####编程思想异同

  • 面向过程 以事件为中心，顺序调用，更注重每个环节的功能。
  • 面向对象 更注重封装和把复杂的问题用现实的思维简化。
  • 面向切面 更注重解耦，在不修改源码前提下，动态为多个业务模块拓展新功能。

  无论面向什么，只是我们用程序去解决实际问题时候的不同角度而已。面向对象将程序划分角色，每个角色负责一部分功能，像我们人类生活一样，抽象化复杂问题的时候，运用这种思想更容易理解；而面向切面，个人理解就是看待的目标不再是将实际问题怎么转化为程序，而是现有的程序，将一个整体的程序按某种逻辑划分几个模块，然后再通过编写 AOP 程序去为划分的模块做拓展。

需要介绍几个概念：

使用

  1. Getting started with AspectJ : https://www.eclipse.org/aspectj/doc/released/progguide/index.html

注意

  • Spring AOP 虽然跟 AspectJ 的有些注解一样，感觉上用法一样，但是 Spring AOP 用的是 JDK 的 DynamicProxy 和 Cglib 实现的代理。是在运行期工作的。AspectJ 虽然也支持运行期AOP，但是多数我们还是用 Java + AspectJ 的注解来实现编译器修改字节码方式来实现 AOP。两者虽然用法类似但是原理不同。
  • 另外整理了一个 Android 的 AspectJ 插件:smiley: ：https://github.com/yangxiaobinhaoshuai/AspectJ4Android

Javassist

资源

  • Github ：https://github.com/jboss-javassist/javassist
  • Project page ：http://www.javassist.org/

Introduction

  • Javassist 含义： Java programming Assistant
  • 官网介绍：是一个通过 Java 来编辑 class bytecode 的 library。通过 Java 在运行期定义新的 class 和在 JVM load class 的时候去修改 class。Javassist 也提供了两层 API ： Source level & byecode level。

ASM

资源

  • 官网 ： https://asm.ow2.io/
Introduction
  • 可以在编译期间，做到对 class 的动态生成、访问、修改等各种操作，并且设计的初衷为尽可能小、高效。
  • 很多知名库都用了 ASM 做底层实现，JDK 内置了 ASM；Cglib 也是基于 ASM 实现的动态代理；包括 Gradle。ASM 可谓大名顶顶。

Usage

ASM 提供了两套 API ，一套基于事件，一套基于对象树。两套 API 又都是基于 Visitor 模式。

Byte buddy

资源

  • Github ：https://github.com/raphw/byte-buddy/
  • project page ：http://bytebuddy.net/#/
Introduction
  • 在运行期修改和生成 calsses 不借助编译器。
  • 允许生成任意 classes ，没有动态代理必须实现接口的限制。
  • 不需要理解 java bytecode 和 class file format。旨在 API 简洁易懂。
