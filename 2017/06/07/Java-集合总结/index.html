通常我们常用的也就ArrayList和HashMap，不过还是需要了解下各个集合特点和底层实现的～

List

特点：

  1. 有序，有角标

  2. 支持for循环和增强for循环

  3. 允许重复的元素

实现类

  1. ArrayList：

    • 底层是数组
    • 查找效率，增加和删除慢，因为这两个操作原理上是数组的拷贝
    • 非线程安全

    • 实现了Serializable，可被用作组件间传递。

      • Serializable 使用了反射来序列化对象，会生成很多对象，尽管只是个标记接口，操作方便，但是性能不如parcelable，推荐使用parcelabe，用插件生成模版代码即可。

  2. LinkedList：

    • 底层是链表

    • 增加和删除效率，查找慢

      • 基本元素是Node，Node封装三个内容：元素值，前一个node和后一个node； 所以理论上一个元素只能知道它前一个和后一个的元素，所以查找起来比较慢。具体如下：这是根据索引查找元素的get方法，中间那句是为了检测索引是否越界，越界就会跑出异常，点进return 的方法中

        ```
        /**

      • Returns the (non-null) Node at the specified element index.
        */
        Node node(int index) {
        // assert isElementIndex(index);

        if (index < (size >> 1)) {

        Node<E> x = first;
for (int i = 0; i < index; i++)
    x = x.next;
return x;

        } else {

        Node<E> x = last;
for (int i = size - 1; i > index; i--)
    x = x.prev;
return x;

        }
        }

        ``` 

可见，先是获     取集合size，然后size右移一位，就是集合中间位置，判断小于这个位置就从前往后遍历，大于这个位置就从后往前遍历，遍历元素必然消耗性能。
  1. Vector：
    • 线程安全的Arraylist
