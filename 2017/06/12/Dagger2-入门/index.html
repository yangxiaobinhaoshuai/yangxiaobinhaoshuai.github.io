庖丁刀：Dagger2

发表于 2017-06-12

Dagger2: Google fork square的项目。
  搜了很多Dagger2的相关材料，发现少得可怜，一直说深入发掘下Dagger2的用法。纸上得来终觉浅，绝知此事要躬行。
  google对项目的描述：”一个Android 和Java的依赖注入器， dagger2 提升为一个编译时依赖注入工具，完全放弃的反射带来的性能上的缺陷，并通过删除传统的ObjectGraph / Injector来改善代码清晰度，取而代之的是@Component接口。“

文档翻译

  1. 首页：dagger是一个全静态的，编译时依赖注解framework，适用于java和Android平台。由Square创建，现在google在维护。
  2. 用户手册：dagger能够代替那些工厂类的实例化注入代码，能够让你专注逻辑，声明注入对象，满足条件即可，起到app瘦身作用。通过javax包下的inject注解，让每个类都更加容易测试；使用dagger也更加容易能写出高可复用，高可拓展的模块，更佳解耦，更加容易测试。
  3. 使用
    • @Inject标记你需要dagger为你创造实例的构造器前，dagger会自动获取需要的参数值，然后调用构造器创建实例，当构造器中有参数的时候，dagger会自动帮你创建参数的实例。
    • 如果一个类如果没有被@Inject标记的构造器，dagger是不会为你创造实例的，也就是说如果你想通过@Inject标记成员来实例化成员，必须在成员的构造器也加上@Inject，即便构造器是默认的无参构造器，也要加上@Inject才行。
    • 但是@Inject有三种情况是无效的
      • 接口不行，dagger没有办法帮你寻找制定实现类。
      • 三方的类不行了，我们无法操作三方类的构造器为其添加@Inject注解
      • Configurable objects must be configured!（没理解了这句… ）
    • 对于不能使用@Inject的情况，需要使用一个带有@Provides标记的方法来提供依赖，该方法返回一个需要被注入的类型对象。
    • 所有的provide方法必须在一个被@Module标记的类中
    • 通常来说，带有@Provides标记的方法名都命名为provide前缀，而包含这个方法的类都带有Module的后缀
  4. 构建对象依赖图
    • 被@Inject和@Provides标记的类，和他们的依赖关系，实际上都存储一个叫对象图的概念中，个人理解这个对象图就是component。
    • 存储变量与类实例依赖关系的纽带是“component”，所以我们要在调用注入实例的类中实例化component，通过component.create()获取component实例。
    • 获取实例可以把一个module作为参数传给component，component会寻找module中是否有被@Provides标记的方法，有的话就拿来用做实例返回，没有的话就去寻找被@Inject标记的对应构造器，调用构造器来获取实例。
    • 如果传入module的构造器为默认构造器，或者被@Provides标记的方法都是static的，构建component就不需要通过builder来传入module实例，直接调用Component.create（）即可。
    • 如果component是内部接口的话，编译生成的相应接口命名会以外部类开头
  5. component 范围
    • 限于Android 应用内存，对于通过dagger大量实例化对象也是耗费资源的。比如，大量的实例单例对象，我们应该通过@Scope来控制公用对象来节省内存开销。
    • 被@Singleton修饰的component，是单例的，多线程共享同一资源。

具体使用

dagger2函数库使用

注解

@Inject

标记声明的构造器，dagger会自动寻找component中有没有@provides标记的方法提供实例，如果没有就去寻找声明对象的构造器是否被@Inject标记，如果是，就实例化这个对象，当然如果有参数的化，dagger会先实例参数，传入参数。

@Component

个人理解就是所有实例化都是由component的实例来完成的，所以我需要在程序开始的时候构建component实例，让dagger完成实例化。有的需要传入module来构建component，但是都需要传入需要在哪个类中实例化的对象（this).

@Module

当三方库我们没有办法用@Inject来标记对象构造器的时候，或者一些接口，没有构造器，dagger提供了通过@provides标记方法返回实例，来弥补@Inject的不足，但是@provides标记的方法都要在@Module的class中。
